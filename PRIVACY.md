# AI Codegen Guard Privacy & Code Security Protocol 🛡️

The ultimate priority of any development-focused software is the **strict sovereign protection of intellectual property, code assets, and raw proprietary logic**. 

Your codebase represents your team's competitive moat. For this reason, the **ProjectGuard AI** team adheres to a strict "Zero-Trust, Zero-Transmission" protocol. We make the following binding privacy and data-handling commitments to all users and enterprise subscribers:

---

## 🔒 1. 100% Local, Client-Side AST Execution

- **No Remote Processing**: All calculations, abstract syntax tree (AST) token parsing, circular dependency audits, physical metric scans, and D3 relationship visualization charts are computed **entirely within your browser's local sandbox or local execution container**.
- **No Source Code Harvesting**: Your source files, file structures, custom implementation names, and architecture specifications are **absolutely never** uploaded, shipped, cached, or transferred to any centralized remote database or remote monitoring server controlled by our team or third parties.

---

## 🌍 2. Transparent Large Language Model (LLM) Communication

When you explicitly trigger and prompt **AI Autopilot Refactoring / Architecture Decoding (Auto-Fix Schemes)**:
1. The tool securely packages **only the specific target file context** that violated your contract, along with the precise error report and surrounding module metadata.
2. No unrelated system directories, global environment files, database records, custom secrets, or independent project assets are ever leaked, read, or added to the communication payload.
3. This payload is transmitted over a secure, encrypted HTTPS protocol to the endpoint you explicitly configure (e.g., your direct Google Gemini API instance or your company's secure custom LLM gateway).
4. Since this transaction bypasses intermediate indexing servers, your code context remains entirely safe from model retraining pools, unless your custom LLM provider states otherwise.

---

## 🏗️ 3. Auditability and Transparency

We maintain a transparent structure. Developers and operations officers are encouraged to audit the front-facing build bundles and network activity indicators inside the running console of AST.Guard. There are no silent background trackers, telemetry pingbacks, or third-party usage-mining widgets loaded into the environment.

---

## ⚖️ 4. Disclaimer of Responsibility

AST.Guard acts as a local static analysis instrument. We are not liable for security breaches resulting from malware in the user's local operating system environment, compromises of the user's custom API key vaults, or man-in-the-middle exploits occurring through unsafe VPN configurations or unauthorized model proxies. Always keep your API keys stored securely in safe system-level environment variables only.

For specialized compliance questions, enterprise deployments, or custom self-contained Docker implementations, please contact the product success desk under your purchased product license seat index.
